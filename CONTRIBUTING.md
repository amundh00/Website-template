# Contributing to Website Template

This is a template repository meant to be forked and customized for your own projects. However, if you find improvements that would benefit others, contributions are welcome!

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists
2. Create a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Include expected vs actual behavior

### Making Changes

1. Fork the repository
2. Create a new branch (`git checkout -b feature/improvement`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

### Code Style

- Follow the existing code style
- Use TypeScript for type safety
- Add comments for complex logic
- Keep components small and focused
- Write descriptive variable and function names

### Testing Checklist

Before submitting:

- [ ] Code builds without errors (`npm run build`)
- [ ] No linting errors (`npm run lint`)
- [ ] All existing features still work
- [ ] New features are documented
- [ ] Environment variables are documented if added

### Adding New Features

When adding new features:

1. Update relevant documentation
2. Add examples if applicable
3. Keep dependencies minimal
4. Consider backwards compatibility
5. Test with both dev and production builds

### Documentation

- Update README.md for major changes
- Update SETUP.md for setup-related changes
- Add inline comments for complex code
- Include JSDoc comments for utilities and functions

## What We're Looking For

Useful additions:

- Bug fixes
- Performance improvements
- Better error handling
- Additional utility functions
- More UI components
- Better TypeScript types
- Documentation improvements

Please avoid:

- Breaking changes without discussion
- Adding heavy dependencies
- Removing existing features
- Changing project structure significantly

## Questions?

Feel free to open an issue for discussion before starting major work.
